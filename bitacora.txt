PASO 5 — ARCHIVOS DE CONFIGURACIÓN CREADOS

.prettierrc
     Configuración de formato automático de código:
     - printWidth: 90 caracteres por línea
     - tabWidth: 2 espacios
     - singleQuote: comillas simples
     - trailingComma: coma final en todos los casos
     - endOfLine: lf (saltos de línea Unix)

.eslintrc.json
     Configuración de estándares de codificación TypeScript:
     - Parser: @typescript-eslint/parser
     - Extiende: eslint:recommended + @typescript-eslint/recommended
     - Reglas: no-explicit-any, no-unused-vars, import/order
     - ignorePatterns: jest.config.js (archivos de configuración JS)
     - Override para archivos .test.ts sin project TypeScript

tsconfig.json
     Configuración del compilador TypeScript:
     - target: ES2020
     - strict: true 
     - outDir: ./dist 
     - rootDir: ./src 
     - types: ["jest", "node"]
     - allowJs: true

jest.config.js
     Configuración de pruebas unitarias:
     - preset: ts-jest
     - testEnvironment: node
     - testMatch: archivos .test.ts

.gitignore
     Archivos ignorados por Git:
     node_modules/, dist/, .env, *.log, coverage/

PASO 8 — CONFIGURACIÓN DE HUSKY

Husky inicializado:
     npx husky-init && npm install

Hook pre-commit configurado (.husky/pre-commit):
     Ejecuta lint-staged antes de cada commit.
     Si ESLint o Prettier fallan, el commit es bloqueado.

Hook pre-push configurado (.husky/pre-push):
     Ejecuta npm test antes de cada push.
     Si los tests fallan, el push es bloqueado.

PROBLEMA ENCONTRADO (PASO 8):
     Al hacer push, TypeScript no reconocía describe/it/expect
     de Jest porque faltaban los tipos en tsconfig.json.
     SOLUCIÓN: Se agregó "types": ["jest", "node"] en tsconfig.json
     y se verificó que @types/jest estuviera instalado.

PROBLEMAS ENCONTRADOS Y SOLUCIONES


PROBLEMA 1:
  ESLint v9 no reconoce el formato .eslintrc.json
  ERROR: "ESLint couldn't find an eslint.config.(js|mjs|cjs) file"
  CAUSA: ESLint v9 cambió el formato de configuración por defecto
  SOLUCIÓN: Degradar ESLint a v8 con:
            npm uninstall eslint
            npm install --save-dev eslint@8 ...

PROBLEMA 2:
  ESLint intentaba analizar jest.config.js con tsconfig.json
  ERROR: "TSConfig does not include this file"
  CAUSA: parserOptions.project aplica a todos los archivos .js
  SOLUCIÓN: Agregar "ignorePatterns": ["jest.config.js"] en
            .eslintrc.json y override para archivos .test.ts

PROBLEMA 3:
  TypeScript no reconoce describe/it/expect en archivos de test
  ERROR: "Cannot find name 'describe'. TS2593"
  CAUSA: tsconfig.json no incluía los tipos de Jest
  SOLUCIÓN: Agregar "types": ["jest", "node"] en tsconfig.json
